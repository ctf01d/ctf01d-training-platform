# Хранилище архивов сервисов

- Директория по умолчанию: `Rails.root/storage/services`.
- Можно переопределить через переменную окружения `SERVICES_STORAGE_DIR` (абсолютный путь).
- Требования: каталог должен существовать или быть создаваемым процессом приложения и быть доступным на запись.

Варианты (Docker):

- Docker Compose (готово из коробки): в `docker-compose.yml` смонтирован volume `services-data` в `/data/services`, а в контейнер передана `SERVICES_STORAGE_DIR=/data/services`. При первом старте права будут корректными.
- Bind‑mount с хоста: создайте папку и смонтируйте её в контейнер. Пример:

  SERVICES_STORAGE_DIR=/data/services
  -v /srv/ctf01d/services:/data/services

  Убедитесь, что пользователь внутри контейнера (UID 1000) имеет права на запись в эту папку (на хосте: `chown -R 1000:1000 /srv/ctf01d/services`).

Если каталог недоступен на запись, UI покажет понятную ошибку вместо 500.
